# SmartFlex: Smart Home Energy Optimisation

**Maximum Comfort, Minimal Charge**

A Deep Reinforcement Learning system that autonomously manages a UK home's solar PV, battery storage, and HVAC (heating/cooling) to minimise energy costs while maintaining occupant comfort.

## Project Overview

This project uses **Soft Actor-Critic (SAC)**, an off-policy deep RL algorithm, to learn optimal control policies for residential energy management. The agent is trained in a custom Gymnasium simulation environment that models:

- **Solar PV generation** with realistic UK irradiance patterns and cloud cover
- **Battery storage** (10 kWh) with round-trip efficiency losses
- **HVAC system** with heating and cooling modes (heat pump COP model)
- **Time-of-use electricity pricing** inspired by Octopus Agile tariffs
- **Stochastic scenarios** across four UK seasons with variable weather, occupancy, and price events

The trained agent is benchmarked against three baselines: a naive thermostat, a rule-based controller, and a random policy.

## Usage

### 1. Train the SAC agent

```bash
python train.py
```

Training runs for ~1.5M steps (~15,000 episodes). Saves:
- `best_model_actor.pth` — best model by validation reward
- `sac_final_actor.pth` — final model after all training
- `training_log_*.csv` — per-episode metrics
- `validation_log_*.csv` — periodic validation results

### 2. Evaluate against baselines

```bash
python test_and_evaluate.py
```

Runs 500 episodes per policy on identical scenarios. Outputs:
- `eval_results_*.csv` — aggregate results with per-season data
- `trajectories_*.csv` — per-step data for trajectory plots
- Statistical summary with 95% CIs and Cohen's d effect sizes

### 3. Generate report figures

```bash
python plot_results.py
```

Produces all plots in the `plots/` directory.

### 4. Interactive dashboard

```bash
streamlit run dashboard.py
```

Opens a web-based dashboard for exploring agent behaviour.

### 5. Run tests

```bash
python -m pytest test_env.py -v
```

## Key Results

## Technical Details

- **Algorithm**: Soft Actor-Critic (Haarnoja et al., 2018) with automatic entropy tuning
- **State space**: 14-dimensional (SOC, solar, price, temperatures, time encoding, occupancy, one-hot season, cloud cover, base load)
- **Action space**: 2-dimensional continuous [-1, 1] (battery charge/discharge, HVAC heating/cooling)
- **Training**: 1.5M steps, 128 batch size, cosine LR annealing, 500K replay buffer
